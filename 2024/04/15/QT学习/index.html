理论：

  • Qt 是一个跨平台的 C++开发库，主要用来开发图形用户界面（Graphical User Interface，简称 GUI）程序。

  • C语言是一种面向过程的编程语言，C++是一种面向对象的编程语言。

  • C++通过 public、protected、private 三个关键字来控制成员变量和成员函数的访问权限，它们分别表示公有的、受保护的、私有的，被称为成员访问限定符

    • 不写成员访问限定符时，默认privite属性。

  • 构造函数：定义了一个名称和类名相同，没有返回值的函数，这个函数称为构造函数。构造函数的特殊之处在于，它会在类实例化的时候被调用。

  • 析构函数：？？对象实例化的时候，会调用构造函数，而对象销毁的时候，就会调用析构函数。

  • 现有的类称为“基类”，继承实现的新类称为“派生类”。

  • 类的继承总结：

    • 公有继承时基类中各成员属性保持不变，基类中private成员被隐藏。派生类的成员只能访问基类中的public/protected成员，而不能访问private成员；派生类的对象只能访问基类中的public成员。
    • 保护继承时基类中各成员属性均变为protected，并且基类中private成员被隐藏。派生类的成员只能访问基类中的public/protected成员，而不能访问 private 成员；派生类的对象不能访问基类中的任何的成员
    • 私有继承时基类中各成员属性均变为private，并且基类中private成员被隐藏。派生类的成员也只能访问基类中的public/protected成员，而不能访问private成员；派生类的对象不能访问基类中的任何的成员。

  • 在同一个作用域内，声明几个功能类似的同名函数，并且这些同名函数的参数个数、参数类型或者参数顺序不同，或者函数返回类型不同，那么就叫函数重载。

  • 栈（Stack）和堆（Heap）是计算机内存中两种不同的分配方式，用于存储程序运行时的数据。它们有着不同的特点和用途。

    1. 分配方式：
      • 栈是一种线性结构，数据的存储和访问遵循后进先出（LIFO）的原则。栈中的内存空间是连续的，由系统自动分配和释放。
      • 堆是一种动态内存分配区域，数据的存储和访问没有特定的顺序。堆中的内存空间是离散的，需要程序员手动管理内存的分配和释放。
    2. 分配速度：
      • 栈的分配速度比堆快，因为栈中的内存分配是由系统自动管理，不需要额外的分配和释放操作。
      • 堆的分配速度比栈慢，因为堆中的内存分配需要动态地搜索可用的内存块，并进行相应的内存管理操作。
    3. 分配大小：
      • 栈的大小通常受限于系统设置的栈大小，一般较小。
      • 堆的大小通常受限于系统可用的虚拟内存大小，一般较大。
    4. 生命周期：
      • 栈中的数据的生命周期通常是在函数执行期间，当函数执行结束时，栈中的数据会自动被销毁。
      • 堆中的数据的生命周期由程序员手动管理，需要显式地释放内存空间，否则可能会导致内存泄漏。
    5. 使用场景：
      • 栈适用于存储函数的局部变量、函数的参数以及函数的返回地址等，以及执行函数调用时的上下文信息。
      • 堆适用于动态地分配内存空间，例如创建对象、数组和其他动态数据结构，以及在程序运行时需要动态分配内存的情况下。

    总的来说，栈和堆是两种不同的内存分配方式，栈是一种自动管理的内存区域，适用于存储局部变量和函数调用信息，而堆是一种手动管理的内存区域，适用于动态分配和释放内存空间。在程序设计中，需要根据实际情况选择合适的内存分配方式，以提高程序的性能和可靠性。

  • QWidget 是 Qt 中用于创建用户界面的基础类，通过它可以创建各种窗口部件，并实现丰富的界面功能。在 Qt 应用程序中，几乎所有的窗口部件都是从 QWidget 派生而来的。

  • QT文件命名 一般是小写
